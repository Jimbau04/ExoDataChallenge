Visión del proyecto

Objetivo central: Construir un sistema de clustering que descubra la "taxonomía natural" de los exoplanetas conocidos, más allá de las categorías clásicas. El diferencial está en la interpretación astrofísica profunda y en presentar resultados como una narrativa científica coherente y visualmente impactante.
🎯 Problema a resolver

El NASA Exoplanet Archive contiene 6,147 exoplanetas con datos heterogéneos, múltiples valores nulos y sesgos de detección. La tarea es agruparlos de forma significativa y traducir esos grupos en conocimiento astrofísico real.

🏆 Criterio de éxito

Clusters que sean estadísticamente válidos (métricas internas), físicamente interpretables (alineados con tipologías conocidas: Hot Jupiters, super-Tierras, etc.) y presentados con narrativa científica clara.

Diferenciadores clave (por qué ganaremos)

La convocatoria puntúa la interpretación astrofísica con 30 puntos — el criterio individual más alto. La mayoría de equipos se quedará en K-Means + Silhouette Score. Nosotros iremos mucho más lejos.
🧬
Multi-algoritmo comparado
Correremos K-Means, DBSCAN, Gaussian Mixture Models y Clustering Jerárquico. Elegiremos el mejor por métricas, no por conveniencia. Eso muestra solidez metodológica.
🌌
Corrección de sesgo observacional
Analizaremos explícitamente los sesgos por método de detección (tránsitos vs. velocidad radial) y corregiremos su efecto en la distribución de clusters. Nadie más lo hará.
🪐
Habitabilidad como eje narrativo
Usaremos pl_insol y pl_eqt para identificar candidatos en zona habitable dentro de los clusters. Un resultado con impacto emocional y científico para el jurado.
📐
Ingeniería de features avanzada
Crearemos features derivados: densidad calculada, relación masa-radio, índice de insolación, ratio periodo/eje mayor (verificación de Kepler). Features que los demás no generarán.
🔭
Imputación científica (no estadística)
Los valores nulos se imputarán usando relaciones físicas reales (relación masa-radio de Zeng & Sasselov), no con medias o medianas. Esto es lo que haría un astrofísico.
🎨
Visualización nivel publicación
Diagrama de Hertzsprung-Russell, diagrama masa-radio con zonas coloreadas por composición, scatter plots en escala logarítmica. Visualizaciones que parecen de una revista científica.

Features derivados (ingeniería de características)

📊 Features a crear
  • pl_dens_calc — densidad calculada si radio y masa disponibles (g/cm³)
  • log_period — log10(pl_orbper) — lineariza la distribución sesgada
  • log_rade — log10(pl_rade) — igual justificación
  • log_bmasse — log10(pl_bmasse) — igual
  • mass_radius_ratio — proxy de composición (alta → rocoso)
  • kepler_check — verifica T² ∝ a³ (validación de consistencia)
  • hz_flag — 0.2 ≤ pl_insol ≤ 1.7 → zona habitable conservadora
🔧 Estrategia de nulos
  • Si radio conocido pero masa no: usar relación Zeng & Sasselov (2016) para estimar masa según tipo (rocoso vs gaseoso)
  • Si masa conocida pero radio no: invertir relación M-R
  • Si pl_eqt nulo pero pl_insol disponible: calcular T_eq = 278.5 × (L*/L_sol)^0.25 × (a/AU)^-0.5
  • Variables con >60% nulos: excluir del clustering, incluir solo en análisis descriptivo
  • Nunca imputar con media/mediana — siempre registrar qué fue imputado con un flag

Paso 1–3: Ingesta, EDA y Feature Engineering

# 1. Leer CSV saltando líneas de comentario (#)
import pandas as pd
df = pd.read_csv('PSCompPars_2026.csv', comment='#')

# 2. Inspección inicial
print(df.shape)  # (6147, 320)
df.info()
df.describe()

# 3. Completitud por columna
completeness = (df.notna().sum() / len(df) * 100).sort_values(ascending=False)

# 4. Features logarítmicos (distribuciones con cola larga)
import numpy as np
df['log_period'] = np.log10(df['pl_orbper'].clip(lower=0.01))
df['log_rade']   = np.log10(df['pl_rade'].clip(lower=0.01))
df['log_bmasse'] = np.log10(df['pl_bmasse'].clip(lower=0.01))
df['log_smax']   = np.log10(df['pl_orbsmax'].clip(lower=0.0001))

# 5. Densidad calculada
df['pl_dens_calc'] = df.apply(
    lambda r: (r['pl_bmasse'] * 5.972e24) /
              ((4/3) * np.pi * (r['pl_rade'] * 6.371e6)**3) / 1000
    if pd.notna(r['pl_bmasse']) and pd.notna(r['pl_rade']) else np.nan, axis=1)

# 6. Flag zona habitable conservadora (0.2 ≤ insol ≤ 1.7)
df['hz_flag'] = df['pl_insol'].between(0.2, 1.7).astype(int)

Paso 4–5: Escalado y Reducción de Dimensionalidad

from sklearn.preprocessing import RobustScaler
from sklearn.decomposition import PCA
from sklearn.manifold import TSNE
import umap  # pip install umap-learn

# Features seleccionadas para clustering (alta completitud + relevancia física)
FEATURES = ['log_rade','log_bmasse','log_period','log_smax','pl_eqt',
            'st_teff','st_met','sy_pnum']

df_clean = df[FEATURES].dropna()  # guardar índice para merge posterior
print(f"Filas para clustering: {len(df_clean)}")

# RobustScaler: mejor que StandardScaler para datos astronómicos con outliers
scaler = RobustScaler()
X = scaler.fit_transform(df_clean)

# PCA para clustering (mantener 95% varianza)
pca = PCA(n_components=0.95, random_state=42)
X_pca = pca.fit_transform(X)
print(f"Componentes PCA: {X_pca.shape[1]}")

# t-SNE para VISUALIZACIÓN (no para clustering)
tsne = TSNE(n_components=2, perplexity=40, random_state=42, n_jobs=-1)
X_tsne = tsne.fit_transform(X)

# UMAP como alternativa más rápida
reducer = umap.UMAP(n_components=2, random_state=42)
X_umap = reducer.fit_transform(X)

Paso 6: Cuatro algoritmos de clustering

K-Means (baseline)

Probar K de 3 a 10. Usar método del codo + Silhouette para elegir K óptima. Ventaja: interpretable. Limitación: asume clusters esféricos.

from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

scores = {}
for k in range(3, 11):
    km = KMeans(n_clusters=k, n_init=20, random_state=42)
    labels = km.fit_predict(X_pca)
    scores[k] = silhouette_score(X_pca, labels)
K_optimal = max(scores, key=scores.get)
DBSCAN (detección de outliers)

Ideal para encontrar planetas "anómalos" (detección directa, planetas libres, circumbinarios). El ruido (-1) es información astrofísica valiosa.

from sklearn.cluster import DBSCAN
from sklearn.neighbors import NearestNeighbors

# Calcular epsilon óptimo con k-dist graph
nbrs = NearestNeighbors(n_neighbors=5).fit(X_pca)
distances, _ = nbrs.kneighbors(X_pca)
eps_opt = np.percentile(distances[:,4], 90)

db = DBSCAN(eps=eps_opt, min_samples=10)
db_labels = db.fit_predict(X_pca)
Gaussian Mixture Models

Clustering probabilístico. Permite clusters elípticos y superpuestos. Ideal para transiciones graduales entre tipos planetarios (sub-Neptuno ↔ Neptuno).

from sklearn.mixture import GaussianMixture

# BIC para seleccionar número óptimo de componentes
bic = []
for k in range(3, 11):
    gmm = GaussianMixture(n_components=k, covariance_type='full', random_state=42)
    gmm.fit(X_pca)
    bic.append(gmm.bic(X_pca))
K_gmm = range(3,11)[np.argmin(bic)]
Clustering Jerárquico

Genera un dendrograma que muestra relaciones entre tipos. Excelente para la presentación visual. No requiere K de antemano.

from scipy.cluster.hierarchy import dendrogram, linkage, fcluster
from scipy.spatial.distance import pdist

# Ward minimiza varianza intra-cluster
Z = linkage(X_pca, method='ward')
# Cortar en N clusters (determinado por análisis previo)
hier_labels = fcluster(Z, t=K_optimal, criterion='maxclust')

Análisis de habitabilidad — el diferenciador narrativo

🌍 Zona habitable conservadora

Planetas con pl_insol entre 0.2 y 1.7 veces el flujo terrestre. Filtrar además por radio < 2 R⊕ (posiblemente rocoso). Listar los top 5 candidatos con nombre, sistema y temperatura de equilibrio.

💧 Condiciones para agua líquida

Temperatura de equilibrio entre 200 K y 320 K (con albedo ~0.3). Estos planetas son el "gancho" emocional de la presentación: ¿podría haber vida allí?

Verificación del "desierto de Neptuno"

Un hallazgo notable que podemos destacar: los exoplanetas con periodos muy cortos (<3 días) y radios intermedios (2–4 R⊕) son extremadamente raros — el desierto de Neptuno. Si nuestro clustering lo captura como una zona de baja densidad entre clusters, eso es un resultado astrofísicamente significativo que el jurado reconocerá.

Visualizaciones planificadas

1. Diagrama masa-radio (el más importante)

Scatter log-log de masa vs. radio, coloreado por cluster. Superponer líneas de composición teórica (hierro puro, roca, agua, gas). Este diagrama es el "resultado central" del proyecto.

2. t-SNE / UMAP 2D coloreado

Espacio reducido 2D con puntos coloreados por cluster. Repetir coloreado por método de detección para mostrar análisis de sesgo. Mostrar ambos lado a lado.

3. Diagrama periodo-radio (Kepler/Fulton)

Scatter de periodo orbital vs. radio, coloreado por cluster. Muestra el desierto de Neptuno y la brecha de radios de Fulton (~1.8 R⊕). Resultado astrofísico de alto impacto.

4. Diagrama H-R extendido

Temperatura efectiva estelar vs. luminosidad (o radio) de la estrella, coloreado por tipo de planeta en el sistema. Muestra qué estrellas hospedan qué tipos de planetas.

5. Mapa de calor de correlaciones

Correlación entre todas las variables seleccionadas (log-transformadas). Identificar qué features son más informativos para la separación de clusters.

6. Dendrograma jerárquico

Árbol de clustering jerárquico truncado a nivel visual. Muestra la estructura de similitud entre grupos — excelente para la presentación oral.

7. Boxplots por cluster

Distribución de cada variable clave por cluster. Permite cuantificar las diferencias y justificar la interpretación astrofísica con estadísticos descriptivos.

8. Mapa de zona habitable

Scatter de flujo de insolación vs. radio, marcando la zona habitable optimista y conservadora. Identificar los candidatos más prometedores como "Tierra 2.0".

Estructura de la presentación (CRISP-DM)

La presentación debe durar entre 10–15 minutos. Cada sección de CRISP-DM es un "acto" de la narrativa. El hilo conductor: "¿Cómo serán los mundos que orbitan otras estrellas?"
Slide 1–2 · Apertura (1 min)
🚀 Hook emocional
Foto de exoplaneta real / imagen artística. Pregunta: "¿Están solos en el universo?". Presentar el reto y el dataset. Cifra impactante: 6,147 mundos confirmados fuera de nuestro sistema solar.
Slide 3–4 · CRISP-DM Fase 1-2 (2 min)
📚 Comprensión del problema y los datos
Explicar qué es PSCompPars. Mostrar el mapa de completitud (heatmap de nulos). Introducir los sesgos observacionales con un gráfico de pastel por método de detección.
Slide 5–7 · CRISP-DM Fase 3 (3 min)
🔧 Preparación de datos e ingeniería de features
Mostrar la estrategia de imputación científica (relación M-R). Explicar los 7 features derivados. Mostrar las distribuciones antes/después de transformación logarítmica. Decisión de variables finales para clustering.
Slide 8–10 · CRISP-DM Fase 4 (3 min)
🤖 Modelado y selección de algoritmo
Presentar los 4 algoritmos con sus resultados. Tabla comparativa de métricas. Justificar la elección del algoritmo final. Mostrar el t-SNE/UMAP coloreado por cluster. Dendrograma.

🪐 Interpretación astrofísica — el corazón del proyecto
Un slide por cluster principal. Diagrama masa-radio con zonas de composición. Identificar Hot Jupiters, super-Tierras, etc. Mostrar el desierto de Neptuno. Presentar los 5 candidatos en zona habitable.
Slide 15–16 · Cierre (1 min)
🌟 Conclusiones y propuestas
Limitaciones (sesgos, nulos, consistencia de datos). Propuesta de mejora: incluir datos espectrales de atmósferas (pl_ntranspec). Aplicación: priorizar targets para futuros telescopios (JWST, PLATO). Mensaje final emotivo.

Riesgos y mitigaciones

RiesgoProbabilidadImpactoMitigación
Muchas variables con nulos no permiten suficientes filas para clustering Media Alto Usar solo variables con >70% completitud. Tener 2 sets de features: "completo" y "reducido".
